Mandatory Tech Stack & Architecture

The application is built on the robust and highly scalable MERN stack, decoupling the frontend client from the backend REST API servers, utilizing industry standard best-practices for stateless session validation and secure AI API proxying.

Frontend - React.js (Vite)

Constructed single-page components utilizing fast ESM modules compiled under Vite, featuring React Router DOM client routing and React Context-style states.

Styling - Tailwind CSS

Incorporates modern, warm sand and emerald-themed glassmorphism styled cleanly with optimized utilities. Fully responsive for Mobile, Tablet, and Desktop screen widths.

Backend - Node.js + Express.js

Asynchronous JavaScript event loop server registering clean JSON endpoint routing, security headers (helmet), and Cross-Origin Resource Sharing (CORS).

Database - MongoDB Atlas

Mongoose ORM schemas mapping User profiles and Opportunity documents. Hosted fully on a multi-region MongoDB Atlas cloud database cluster.

AI Engine - Gemini 2.5 Flash

Google Generative AI SDK used securely on the backend server. Communicates dynamically via user prompt context to produce short, highly helpful student suggestions.

Session Security - JWT & bcrypt

Passwords hashed using 10 rounds of high entropy salt factors before entering the database. Auth gates mapped using cryptographically signed JSON Web Tokens.

Challenges Faced & Solutions Implemented

Transitioning the codebase from a simple local mockup to a highly modular, secure, production-grade MERN stack application presented several challenges, ranging from database indexing structures to secure environment deployments. Mapped below are all key issues resolved during integration.

Development Challenges
Issue 01

Static Opportunities Data

Problem

Initially, the dashboard displayed hardcoded opportunities data directly inside Dashboard.jsx, making the application non-scalable and difficult to manage dynamically.

Solution

Replaced the hardcoded array with dynamic data fetched from MongoDB using React useState and useEffect hooks.

Issue 02

Missing Loading State

Problem

The dashboard UI rendered before backend data was successfully fetched, leading to broken UI states, missing indexes, or empty content flashes on slower networks.

Solution

Implemented an overlay loading state (const [loading, setLoading] = useState(true);) that ensures the dashboard UI remains stable and clean until the backend API completes its response.

Issue 03

Hardcoded Apply Button Redirect

Problem

The opportunity cards' Apply button redirected users to a fixed, generic GitHub repository URL instead of custom individual application portals.

Solution

Modified the frontend apply handler to dynamically resolve and redirect to opportunityItem.applyLink in a new tab.

Issue 04

Missing applyLink Field in MongoDB

Problem

The frontend displayed an error "Apply link not available for this opportunity" because preexisting MongoDB Atlas documents lacked the required URL field.

Solution

Updated the MongoDB collection documents and refactored the backend database seeder to store explicit applyLink: "https://..." properties for each opportunity.

Issue 05

Mongoose Schema Update

Problem

The backend Mongoose schema did not originally support storing and validating application URLs, causing writes to reject or silently ignore the parameter.

Solution

Updated the backend Opportunity.js schema model explicitly, marking applyLink as a required schema parameter.

Deployment & Production Issues
Issue 06

Exposed MongoDB Connection String

Problem

The MongoDB connection string containing live username and password credentials was accidentally exposed because the backend .env file was committed to GitHub.

Solution

Immediately registered .env inside the backend's .gitignore file, revoked the exposed credentials on MongoDB Atlas, and generated a new secure API cluster user credential set.

Issue 07

Database Connection Failure (querySrv ENOTFOUND)

Problem

The backend server failed to connect to MongoDB Atlas upon deployment to Render, returning querySrv resolution errors because the password contained special characters (like '@') which corrupted URI connection bounds.

Solution

Encoded the password special characters using standard URI encoding and updated the MongoDB URI configuration accordingly on the Render Dashboard.

Issue 08

Missing Database Name in MongoDB URL

Problem

The MongoDB connection URI lacked an explicit database name at the end of the query string, which caused MongoDB to write records to a generic test database instead of the targeted application collection.

Solution

Updated the URI string template to explicitly route documents to the intended database cluster context: mongodb+srv://username:password@cluster.mongodb.net/student_db.

Issue 09

Incorrect .env Protection Method

Problem

Attempted to manually hide or lock the local backend .env file using Windows PowerShell system attributes rather than using Git control exclusion commands.

Solution

Used a proper repository-level .gitignore file to robustly prevent the Git tracking tree from indexing the file.

Issue 10

Frontend Asset Path Errors (404)

Problem

Images and vector illustrations failed to load after production deployment (returning 404), because Vite automatically renames, hashes, and compiles static assets during builds, breaking hardcoded file paths.

Solution

Updated the codebase to use modern Vite asset imports (import scholarshipImg from "../assets/scholarship.png"), enabling Vite's bundler to securely index and resolve URLs in production.

Issue 11

React Router Deployment Errors

Problem

Navigating directly to frontend routes like `/signup` or refreshing the dashboard showed blank pages, returning MIME type errors or 404s in production on Render.

Solution

Configured rewrite and redirect rules on Render's static site configuration to rewrite all client paths to index.html, allowing React Router to manage paths dynamically.

Issue 12

Missing Frontend Environment Variables

Problem

The Gemini AI chatbot failed to compile or respond in the live deployed website because the public variable VITE_GEMINI_API_KEY was missing from Render's environment settings.

Solution

Refactored the chatbot to query the backend securely via a POST request to /api/chat. This securely reads the key from Render's backend variables, eliminating client-side exposure.

Issue 13

Broken Authentication API Endpoints

Problem

Login and Signup API calls returned 404 errors in production because frontend requests hit incomplete root routes like `/auth/login` instead of `/api/auth/login`.

Solution

Corrected the shared Axios configuration base URL in the frontend (frontend/src/api/axios.js) to explicitly use https://student-opportunity.onrender.com/api.

RESTful API Endpoints Map

The backend exposes a highly secure REST API structure. To prevent the massive security risk of exposing your private Gemini API key, all AI Chat queries are proxied securely through the backend.

Endpoint Route Method Request Body Payload Server Response Output
/api/auth/signup POST { name, email, password, secretQuestion, secretAnswer } Register profile in DB. Returns status 201 Created.
/api/auth/login POST { email, password } Validates credentials. Returns signed session token and user object.
/api/auth/forgot-password POST { email, secretAnswer, newPassword } Resets password string securely if security answer matches.
/api/opportunities GET None Queries and returns all Scholarships, Internships, and Jobs in MongoDB.
/api/chat POST { message, history } Proxies conversation string securely to Gemini, returning AI response safely.

Local Environment Setup Guide

To boot this application locally for verification or further coding cycles, follow these step-by-step instructions in your terminal environment.

1. Backend Setup

Navigate to the server directory, install dependencies, configure environment, and start the node process:

cd backend
npm install
# Create a .env file and add details:
PORT=5000
MONGO_URI=your_mongodb_connection_string
JWT_SECRET=your_jwt_secret_key
GEMINI_API_KEY=your_google_ai_studio_api_key

# Boot server:
npm run dev

2. Frontend Setup

In a separate terminal shell, navigate to the frontend directory, install local packages, and boot up the hot-reloading Vite dev compiler:

cd frontend
npm install
npm run dev

Future Improvements & Roadmap

To transition this functional internship task from a production-ready dashboard into a highly scalable, enterprise-level student career networking ecosystem, the following strategic feature upgrades are proposed:

🕵️‍♂️ Automated Opportunity Scraper

Implement backend cron jobs using headless browser scrapers (Playwright/Puppeteer) to automatically scan official internship, scholarship, and job boards to inject new active listings daily into MongoDB Atlas.

📄 AI Resume Parser & Matcher

Allow students to drag and drop their PDF resumes. The backend extracts skills and project experiences using OCR, running a semantic vector search to recommend positions with the highest match percentages.

🔔 Automated Notification Alerts

Integrate web push and email notification channels (e.g. Nodemailer/SendGrid), alerting students immediately when a new opportunity matching their exact interests and skills is posted.

🏢 Recruiter & HR Posting Portals

Create dedicated corporate role access levels, enabling authenticated HR recruiters to register, log in, manage their company profiles, directly list vacancies, and review applicant resumes.

🗺️ Skill Gap Roadmaps

When a student views a position they don't fully qualify for, the career AI analyzes their missing skills and dynamically compiles a personalized step-by-step learning roadmap with free resources.

📊 ATS Application Tracker

A built-in Kanban application tracking board (Trello-style) where students can track their applications through various pipeline stages: "Applied", "Interviewing", "Offer", and "Declined".
