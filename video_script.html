Scene 1: Introduction & Objective

0:00 - 0:40
Screen Action

"Hello everyone! My name is Kavya Choudhary, and I am excited to walk you through my implementation of the AI-Powered Student Opportunity Dashboard, which I have developed and deployed for the Alphaloop MERN Stack Internship evaluation. Scene 2: Architecture & Tech Stack

0:40 - 1:10
Screen Action
Open VS Code briefly. Point to the file structure showing the separate /backend and /frontend folders to show proper modular organization. Open the package.json file in the backend to show the installed dependencies.
Spoken Narration
"From an architectural standpoint, the project follows a decoupled MERN stack approach. On the frontend, we use React.js compiled under Vite for quick load speeds, styled with Tailwind CSS for curated, responsive branding. On the backend, we run an Express.js server hosted on Node.js, which maps schema models to our cloud-hosted MongoDB Atlas database cluster. We have also integrated the Google Generative AI SDK, using secure backend routes to handle completions so no sensitive API credentials are ever exposed on the client."

Scene 3: Auth Gateways (Signup/Login)

1:10 - 1:50
Screen Action
Go back to the browser and click "Login" to open the login page. Show the forgot-password overlay modal, type in a mock email, and explain the secure credential validation checking process.
Spoken Narration
"Next, let's explore our secure Authentication Module. The platform utilizes stateless JWT authentication. When registering, passwords are encrypted on the backend server using high-entropy hashing via bcryptjs. We also collect a secret recovery question and answer, allowing users to trigger a password reset securely if they ever lose access. Let's log in to the dashboard."

Scene 4: Dashboard, Live Query Filters, and AI Matching

1:50 - 2:50
Screen Action
Log in to show the Dashboard. Point to the analytics cards at the top showing the 6 statistics. Play with the filters (type in a skill, select "Scholarship", select "Remote") and click the "Apply Now" button to show a successful redirect.
Spoken Narration
"Here we are on the main dashboard, which renders all key metrics at a single glance: Total Positions, active **Scholarships**, live **Internships**, available **Jobs**, **Bookmarked** numbers, and **Recently Viewed** tracking slots. All listing cards are queried in real time from our live MongoDB Atlas database. Students can filter opportunities on the fly by category, location, remote work mode, and required skills. Additionally, we have an AI matching recommendation panel at the top. The engine automatically scans the user's profile interests and skills from MongoDB and filters matching database entries in real time, making discovery seamless. Clicking any 'Apply' button immediately opens the specific portal in a new tab."

Scene 5: Secure AI Chatbot Float Widget

2:50 - 3:30
Screen Action
Click the emerald Career AI floating widget at the bottom right. Type in a message: "Tell me about engineering careers". Show the typing animation bubble and let the AI response render live.
Spoken Narration
"Another core feature of this platform is our Career AI floating widget, **EduInfo AI**. Instead of directly invoking the Google AI SDK from the React client (which would expose our API key in production javascript bundles), the chatbot issues a POST request to our secure backend `/api/chat` route. The backend reads the private `GEMINI_API_KEY` from its Render environment settings, contacts the stable next-gen **Gemini 2.5 Flash** model, maintains system instructions to keep answers clear and student-focused, and returns the response securely."

Scene 6: Challenges Faced & Overcoming Them

3:30 - 4:10
Screen Action
Show the project's root folder files and point back to the browser.
Spoken Narration
"Transitioning this project from a local prototype to a production-grade MERN deployment on Render was an outstanding learning experience. I tackled and resolved multiple challenges: First, I replaced static, hardcoded frontend arrays with dynamic queries to MongoDB Atlas, implementing a robust load state overlay. Second, I corrected production routing. I configured rewrite rules on Render to prevent MIME errors on router refreshes, corrected incomplete API endpoints (mapping requests from root `/login` to `/api/auth/login`), and resolved database querySrv network errors on Render by properly URL-encoding password strings. Lastly, I secured the chatbot, moving all API key interactions from the public frontend static bundles into our secure backend proxy route."
